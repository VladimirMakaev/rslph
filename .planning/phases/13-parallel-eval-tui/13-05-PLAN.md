# Phase 13 Plan 05: Plan Command TUI Default

## Context

**Gap:** TUI mode should be default for plan command
**Severity:** Major
**UAT Test:** 4
**Root Cause:** Design decision - `--tui` should become `--no-tui` flag

User feedback: "Make TUI default and change flag to --no-tui to disable"

## Goal

Make TUI mode the default for the plan command, with `--no-tui` flag to disable.

## Tasks

### Task 1: Change PlanArgs --tui to --no-tui

**Files:** `src/cli.rs`

**Changes:**
1. In `PlanArgs` struct, rename `tui` field to `no_tui`
2. Change clap attribute from `--tui` to `--no-tui`
3. Update help text: "Disable TUI mode (use plain output)"

**Before:**
```rust
/// Use TUI mode for streaming output display
#[arg(long)]
pub tui: bool,
```

**After:**
```rust
/// Disable TUI mode (use plain output)
#[arg(long)]
pub no_tui: bool,
```

### Task 2: Invert logic in plan command handler

**Files:** `src/cli.rs` (in run() match arm for Commands::Plan)

**Changes:**
1. Change `args.tui` to `!args.no_tui` when calling `run_plan_command`

**Before:**
```rust
Commands::Plan(args) => {
    ...
    run_plan_command(..., args.tui, ...)
}
```

**After:**
```rust
Commands::Plan(args) => {
    ...
    run_plan_command(..., !args.no_tui, ...)
}
```

### Task 3: Update tests if any reference --tui flag

**Files:** Any test files using `--tui` flag

**Changes:**
1. Search for tests using `--tui` flag
2. Update to use `--no-tui` or remove flag (since TUI is now default)

## Verification

1. `rslph plan --help` shows `--no-tui` flag (not `--tui`)
2. `rslph plan "test"` launches TUI by default
3. `rslph plan "test" --no-tui` uses plain output mode
4. All existing tests pass
